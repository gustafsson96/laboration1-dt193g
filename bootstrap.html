Om Bootstrap

Bootstrap är ett populärt och komponentbaserat ramverk med öppen källkod. Det kommer därmed med fördesignade komponenter som inkluderas med hjälp av klasser, och installeras antingen som en CDN eller som ett paket i någon av de vanliga pakethanterarna (npm, yarn eller RubyGems för att nämna några exempel). Länkar man till Bootstrap via deras CDN är CSS- och JavaScript-koden redan kompilerad, men installeras det som ett paket för att man till exempel vill modifiera komponenter måste man kontrollera att det finns en Sass-kompilerare och Autoprefixer.

All denna information har hämtats från Bootstraps dokumentation och videon ”Fullstacksutveckling med ramverk – Intro + föreläsning 1” av Malin Larsson, lärare på Mittuniversitetet.

Utvärdering översikt

Även om jag inte arbetat med Bootstrap tidigare har jag ändå tagit del av en del videos och kod-exempel där man använt det som ramverk, så det kändes inte helt främmande när jag skulle påbörja arbetet med denna sida. Jag inkluderade Bootstrap med hjälp av deras CDN, vilket var gick väldigt smidigt. Med hjälp av de färdiga komponenterna och dokumentationen som är välstrukturerad och lätt att förstå kändes det som att jag snabbt kunde skapa mycket innehåll på kort tid. Eftersom jag dessutom har erfarenhet av CSS flex och grid sedan tidigare tog det inte alltför lång tid innan strukturen för positionering av innehåll kändes logisk. Jag är dock inte van vid att arbeta mobile-first, vilket är utgångspunkten i Bootstrap, så det påverkade inlärningskurvan för mig en aning.

Utvärdering för- och nackdelar

Det jag uppskattar främst med Bootstrap är hur snabbt det gick att komma i gång och hur lite kod som krävs för att skapa design som ser helt okej ut som minst. Jag upplever även att det är lätt att söka rätt på ny information via dokumentationen, så det känns som att man kan både lära sig och åstadkomma mycket på kort tid. Det är definitivt ett ramverk jag vill arbeta mer med eftersom jag är övertygad om att det kan vara hjälpsamt för många olika typer av projekt i framtiden, inte minst för att skapa prototyper.

Bootstrap är dock inte det ramverk jag hade valt om jag var ute efter att skapa en väldigt specifik eller mer personlig design. Eftersom det kommer med färdiga komponenter kommer de även med ett standard-utseende som kanske inte passar all design. Det finns möjlighet att modifiera Bootstrap-komponenter via SASS/SCSS om man installerar det som ett npm-paket, men om man vet med sig att det kommer krävas mycket modifiering skulle jag personligen hellre välja ett annat ramverk där jag kan styra utseendet från grunden.

Jag är inte heller helt imponerad över den stora mängden div-element som Bootstrap-kod ofta verkar resultera i, främst för att jag tycker att koden ser ostrukturerad och repetitiv ut om man har mycket innehåll som kräver olika behållare. Det är mer av en personlig smaksak kanske, och för projekt där jag vill se resultat snabbt väger fördelarna ändå tyngre än nackdelarna när det kommer till Bootstrap.
