GitHub Pages

Jag använder vanligtvis Netlify för att publicera mina webbsidor, men eftersom jag tidigare funderat på att testa på GitHub Pages kändes det som ett självklart val för denna uppgift. Github Pages är en tjänst från GitHub som direkt från ett repository möjliggör publicering av statiska filer skrivna i HTML, CSS och JavaScript, antingen för ett GitHub-konto (owner.github.io) eller för ett specifikt projekt (owner .github.io/repository-name). Tjänsten är gratis att använda för att publicera ett publikt repository, men för ett privat repository krävs ett betalkonto. Alla som använder GitHub Pages har en skyldighet att följa ”GitHub Terms of Service”.

Det finns möjlighet för en ”build”-process innan publicering om så skulle behövas. Det rekommendera tillvägagångssättet är att använda den statiska webbplatsgeneratorn Jekyll som har inbyggt stöd för GitHub Pages, men man kan även använda andra verktyg för detta om man så önskar. Eftersom jag använder Bootstraps CDN och TailwindCSS CLI (CSS-koden lagras kontinuerligt i output.css) för ramverken i denna lösning har jag dock ingen kod som kräver ett ”build”-kommando för tillfället.

Min upplevelse är att GitHub Pages var väldigt enkelt att använda för att publicera en simpel webbplats skapad i HTML och CSS med Bootstrap och Tailwind CSS som ramverk (och därmed även en del automatiskt inkluderad JS-kod). Det finns dock en del begränsningar som gör att det inte lämpar sig för alla projekt. Det är till exempel inte tänkt att användas för e-handel eller transaktioner av olika slag, och de publicerade webbsidorna har dessutom en storleksgräns på 1GB.

All denna information är hämtad från den officiella dokumentationen för GitHub Pages via GitHub Docs.

Publicering

Denna webbplats har publicerats enligt följande steg (för ett GitHub-konto med engelska som språk):

  1. Navigera till det repository som innehåller koden du vill publicera (eller skapa ett nytt repository).
  2. Klicka på ”Settings” i menyn.
  3. Klicka på ”Pages” under ”Code and automation” i sidomenyn.
  4. Under ”Branch”, välj vilken gren (branch) och vilken mapp webbplatsen ska byggas från (för detta projekt är det satt till ”main” och ”/root”).
  5. Efter en stund dyker länken för den publicerade webbplatsen upp högst upp på sidan. Klicka på ”Visit site” för att navigera dit.
