Tailwind CSS

Om Tailwind CSS

Tailwind CSS är ett så kallat ”utility-first”-ramverk, vilket innebär att styling sker med hjälp av mindre, färdiga klasser som inkluderas direkt i HTML-koden. Dokumentationen för en av de tidigare versionerna av Tailwind CSS (v0.7.4) fastställer att det är ett ramverk utan både fördesignade komponenter och standardtema, vilket innebär att utvecklaren har full kontroll över webbsidans utseende. Det finns dock färdiga stilmallar och kod-block för till exempel menyer och användbara sektioner av olika slag som fördesignats med klasser av Tailwind-teamet. Dessa finns tillgängliga via Tailwind PLUS och kan med enkelhet kopieras och modifieras utefter behov, men enbart ett fåtal är gratis att ta del av. Denna information har hämtats från Tailwind CSS och videon ”Fullstacksutveckling med ramverk – Intro + föreläsning 1” av Malin Larsson, lärare på Mittuniversitetet.

Utvärdering

Efter en del arbete med ramverket Bootstrap kändes inlärningskurvan för Tailwind CSS rätt skarp. Jag började med att läsa dokumentationen och testade successivt att applicera olika klasser på elementen i min kod utefter det jag läste för att se hur designen påverkades. Det kändes inledningsvis ovant att inkludera en stor mängd klasser direkt i HTML-koden för att få till den styling jag önskade eftersom jag är van vid att hålla HTML-kod och CSS-kod separerade. Efter att ha kollat upp en del exempel och testat modifiera färdig kod från TailwindPLUS tog det dock inte allt för lång tid innan jag började uppskatta arbetsprocessen. Även tillägget ”Tailwind CSS IntelliSense” i Visual Studio Code som ger förslag på klasser bidrog till att det kändes lättare.

För- och nackdelar

Det jag upplever är de största fördelarna med Tailwind CSS är att de fördefinierade klasserna erbjuder en stor flexibilitet, och att det (efter att man vant sig vid arbetssättet) är behändigt att kunna hantera styling direkt i HTML-koden. Tack vare en välskriven och strukturerad dokumentation var det dessutom lätt att komma i gång. Jag inkluderade först deras Play CDN i head-avsnittet, men eftersom den inte ger någon möjlighet att kompilera CSS-koden installerade jag sedan Tailwind CLI.

Mitt största problem med Tailwind CSS har hittills varit att det ibland känns som man måste arbeta med onödig upprepning av koden, särskilt när man är van vid att försöka undvika redundans. Detta blev särskilt tydligt då jag arbetade med tabellen längre ned på denna sida och insåg att jag var tvungen att styla varje enskilt th-element och td-element. Det finns färdigdesignade tabeller via tailwindPLUS, men eftersom de är bakom en betalvägg kom jag inte åt dem. Jag testade dessutom att skapa ett separat Vite-projekt och installerade Tailwind som ett plugin för att se om jag kunde hitta någon bättre lösning. Jag testade bland annat att lagra tabell-stylingen i så kallade ”@layer components” i en separat css-fil, vilket fungerade rätt bra. Det kändes dock lite onödigt att ha en så pass liten del av stylingen i en separat fil, så jag kan fortfarande inte helt bestämma mig för vad jag föredrar och bestämde mig för att behålla all styling skapad av mig i HTML-koden för den här sidan. Kanske att det kan vara värt det i större projekt.
